# 展示会タイムトラッカー

展示会の「時間進捗」と「リード数・NA数の達成状況」をリアルタイムで可視化するダッシュボード。
スプレッドシート（Sansan社内限定シート）から自動でデータを取得し、PiP（ピクチャーインピクチャー）で
他の画面に重ねて常時表示できます。

## アーキテクチャ

```
┌─────────────────────┐   <script>タグ(JSONP)    ┌──────────────────────┐
│  GitHub Pages        │ ───────────────────────▶ │  GAS Web App         │
│  index.html (表示)   │   ?sheet=...&callback=fn  │  Code.gs (データAPI) │
│  ・PiP制限なし       │ ◀─────────────────────── │  ・ContentService    │
└─────────────────────┘     fn({...JSONデータ}) ; └──────────┬───────────┘
                                                              │ getActiveSpreadsheet()
                                                   ┌──────────▼───────────┐
                                                   │  スプレッドシート     │
                                                   │  （Sansan組織限定）   │
                                                   └──────────────────────┘
```

- **表示（index.html）** は GitHub Pages でホスト。`script.google.com` の Permissions-Policy に
  縛られないため `documentPictureInPicture` API が使える。
- **データ取得** は GAS Web App（`Code.gs`）。HtmlService をやめ ContentService で JSON / JSONP を返す
  「データ専用 API」。
- GitHub Pages → GAS 間の CORS は **JSONP（`<script>`タグ）方式**で回避。

> ⚠️ **重要（認証の制約と、その回避策）**
> github.io から script.google.com への JSONP は *サードパーティ* リクエストとなり、現在のブラウザは
> サードパーティCookieをブロックします。そのため **ドメイン限定（社内限定）デプロイの GAS は
> 社外から認証が通らず失敗** します。さらに Sansan ドメインのアカウントは管理者ポリシーにより
> Web App を「全員」公開できません。
>
> **回避策：GAS を「個人Googleアカウント」で作る。**
> 対象シートに共有されている個人アカウント（例: 展示会チーム共有の `sansanexposhare@gmail.com`）で
> スクリプトを作成・デプロイします。個人アカウントは管理者ポリシーの対象外なので Web App を
> **「全員（Anyone, even anonymous）」** で公開でき、ログイン不要・社外のレンタルPCからも取得できます。
> スクリプトは `SpreadsheetApp.openById(SPREADSHEET_ID)` でシートを読みます（その個人アカウントが
> シートに共有されていれば読めます）。

## セットアップ

### 1. GAS 側（データ API）

**対象シートに共有されている個人Googleアカウント**（例: `sansanexposhare@gmail.com`）で作成します。
Sansanドメインのアカウントでは「全員」公開できないため、必ず個人アカウントで作業してください。

#### A. スクリプト作成

1. 対象の個人アカウントでログインした状態で <https://script.google.com> を開く →「新しいプロジェクト」
   （※「単体スクリプト」。シートの「拡張機能 > Apps Script」から作るとそのシート所有者に紐づくので注意）
2. `Code.gs` の中身を貼り付け
3. 冒頭の **`var SPREADSHEET_ID = '';`** に対象シートのIDを設定
   （シートURL `…/spreadsheets/d/【ID】/edit` の `【ID】` 部分）
4. プロジェクト設定（⚙️）で「`appsscript.json` を表示」を ON にし、このリポジトリの内容を反映
5. （任意）保護したい場合は `var ACCESS_TOKEN = '';` に合言葉を設定

> clasp を使う場合は、その個人アカウントで `clasp login` → `clasp create --type standalone` →
> `clasp push`。`.clasp.json`（`scriptId`）は Git にコミットしないでください（`.gitignore` 済み）。

#### B. Web App としてデプロイ

GAS エディタ右上「デプロイ > 新しいデプロイ」→ 種類「ウェブアプリ」

| 項目                   | 設定値                                          |
| ---------------------- | ----------------------------------------------- |
| 次のユーザーとして実行 | **自分**（＝この個人アカウント。シート権限を持つ） |
| アクセスできるユーザー | **全員**（＝ Anyone, even anonymous）           |

> `appsscript.json` の `executeAs: USER_DEPLOYING` / `access: ANYONE_ANONYMOUS` がこれに対応します。
> 初回デプロイ時に承認画面が出ます（個人アカウントでシート読み取りを許可）。
> 「全員」が選べない場合は Sansanアカウントでログインしている可能性大 → 個人アカウントか確認。

デプロイ後に発行される **`https://script.google.com/macros/s/＿＿＿/exec`** が GAS URL です。

#### アクセス権限とセキュリティ

「全員」公開のため、**URL を知っていれば誰でも（ログイン不要で）データを取得できます**。
返るデータには担当者名・各人のNA/リード件数が含まれる点に注意してください。
保護したい場合は、合言葉トークンを併用します：

1. `Code.gs` の `var ACCESS_TOKEN = '';` に合言葉を設定（例: `'exhibition-2026-xyz'`）して再デプロイ
2. アプリの「合言葉トークン」欄に同じ値を入力
3. `?token=合言葉` が一致したリクエストにのみデータを返すようになります

> ⚠️ トークンは index.html（クライアント）に埋め込まれます。GitHub Pages を **公開（Public）**で
> 運用するとソースに丸見えになり保護になりません。トークンが実効性を持つのは
> **GitHub Pages を「組織限定(Private Pages)」**にできる場合のみです
> （GitHub Enterprise 等。これにより「普段は社内のみ閲覧／必要時にトークン付きURLを社外共有」が成立）。
> ACCESS_TOKEN を空文字のままにすれば認証チェックは無効（URL のみが秘密）です。

#### 動作確認

ブラウザで以下を開き、JSON が返れば成功（「全員」公開なのでログイン状態は問いません。
シークレットウィンドウなど未ログイン状態で開けると、社外PCでの動作確認になります）：

```
https://script.google.com/macros/s/＿＿＿/exec?sheet=<シート名>
```

JSONP を確認したい場合：

```
https://script.google.com/macros/s/＿＿＿/exec?sheet=<シート名>&callback=test
→  test({...});  が返る
```

### 2. GitHub Pages 側（表示）

1. このリポジトリを GitHub に push
2. リポジトリ Settings > Pages > Source を `main` ブランチ（`/root`）に設定
3. 公開された URL（例: `https://<ユーザー>.github.io/<リポジトリ>/`）を開く

### 3. アプリ上の設定

1. 「⚙️ 基本設定」で会期日数・日程・時間を入力
2. 「データ連携方法」で **🔗 スプレッドシート連携** タブを選択
3. **GAS Web App URL**（上記 `…/exec`）を入力 ※初回のみ。ブラウザに保存され次回自動入力
4. **シート名（タブ名）** を入力（毎回変更可）
5. **合言葉トークン**（GAS側で設定した場合のみ）を入力 ※未設定なら空欄
6. 「🚀 トラッキング開始」→ 60秒ごとに自動同期

## 取得セル仕様（シートのレイアウト）

| データ              | セル                                  |
| ------------------- | ------------------------------------- |
| 1人あたりNA目標     | `B7`（達成者判定のしきい値に使用）    |
| リード目標（日別）  | `B11` / `F11` / `J11`（Day1/2/3）     |
| リード実績（日別）  | `C11` / `G11` / `K11`                 |
| NA目標（日別）      | `B14` / `F14` / `J14`                 |
| NA実績（日別）      | `C14` / `G14` / `K14`                 |
| 担当者名            | `B29〜`                               |
| NA個人実績          | `J29〜`                               |

担当者リストは「`合計`」行を Day の区切りとして読み取ります（最大3日分）。
「`社員計`」以降〜次の「`合計`」までの行（コンパニオン等）は集計から除外します。

## PiP（ピクチャーインピクチャー）について

- 対応ブラウザ（Chrome/Edge 等）では `documentPictureInPicture` API でフローティング表示。
- 非対応・ブロック時は `window.open` のポップアップ表示に自動フォールバック。
- GitHub Pages 配信なら GAS の Permissions-Policy 制限を受けないため、Document PiP がそのまま動作します。

## トラブルシューティング

| 症状                                | 確認ポイント                                                         |
| ----------------------------------- | -------------------------------------------------------------------- |
| 「読み込み失敗」「タイムアウト」（特に社外） | デプロイのアクセス権限が **「全員」** になっているか（ドメイン限定だと社外で失敗） |
| 「読み込み失敗」と出る              | GAS URL（`…/exec`）が正しいか。末尾が `/exec` か（`/dev` ではない）  |
| 「認証エラー: token が一致しません」 | アプリの合言葉トークンと `Code.gs` の `ACCESS_TOKEN` が一致しているか  |
| データが空 / 0 のまま               | シート名（タブ名）が完全一致しているか                               |
| 達成者が表示されない                | `B7`（NA目標）が入力されているか                                     |
| デプロイ更新が反映されない          | コード変更後は「新しいバージョン」で再デプロイ（URLは維持される）    |
